1. Open terminal on your local machine

2. Navigate to the project:
   cd /path/to/kwb-salesforce-load-system

3. Deploy to your dev org:
   sfdx force:source:deploy -u kwb-dev --sourcepath force-app

   Setup → Objects and Fields → Custom Objects

   Edit: force-app/main/default/classes/LoadsmartPoller.cls

   Developer Console → Execute Anonymous:

   System.debug(data.legalName);

================================================================================
ARCHITECTURE
================================================================================

Data Flow:
  Loadsmart API → Polling Job → Salesforce Load Creation → User Assigns Carrier

Polling:
  - Runs every 5-10 minutes (you configure frequency)
  - Fetches shipments updated in last 15 minutes
  - Auto-creates Load records with all data
  - Prevents duplicates via external ID (Loadsmart_Shipment_ID__c)

FMCSA Integration:
  - Searchable by MC# or USDOT#
  - Real-time lookup (Federal Motor Carrier Safety Administration)
  - Returns: Legal name, address, DOT status, insurance status

Post-Back (Phase 2):
  - Included but DISABLED (don't sync to Loadsmart yet)
  - Code ready for testing once polling is stable

================================================================================
TROUBLESHOOTING
================================================================================

"Deploy failed": Check DEPLOYMENT.md for troubleshooting section

"Polling shows 0 loads": 
  - Add Loadsmart credentials first
  - Check API is accessible from your org
  - Run manually: Database.executeBatch(new LoadsmartPoller());

"FMCSA lookup returns null":
  - Verify MC# exists in federal database
  - Test with real carrier: MC 1776323 (KWB Logistics)

"Loadsmart 401 error":
  - Verify Client ID and Secret are correct
  - Credentials have proper scopes/permissions
